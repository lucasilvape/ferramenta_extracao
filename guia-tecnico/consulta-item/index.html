    🔍 Consulta Item

    O módulo src/pages/consulta_apolice.py implementa a classe ConsultaApolice, responsável por toda a lógica da página de consulta de apólices via API externa.

    🎯 Objetivo da Página

    Permitir que o usuário consulte informações de apólices diretamente de uma API, filtrando por Número de Bilhete ou CPF/CNPJ, com suporte aos ambientes de Homologação e Produção. Os resultados são exibidos em uma tabela interativa e podem ser exportados em Excel.

    🧩 Estrutura da Classe

    class ConsultaApolice:
    def __init__(self, main_window):
        self.main_window = main_window
        self.ui = main_window
        self.df_atual = None           # DataFrame com o resultado da última consulta
        self.thread_consulta = None    # Instância da ConsultaApoliceThread
        self.progress_dialog = None    # QProgressDialog exibido durante a busca
        self.inicio_consulta_segundos = 0  # Marca de tempo para cálculo de duração

    🖥️ Widgets Controlados

    Widget Tipo Função
    le_bilhete_sinistro QLineEdit Campo de entrada do número do bilhete
    le_cpf_sinistro QLineEdit Campo de entrada do CPF/CNPJ
    rb_homolog QRadioButton Seleciona ambiente de Homologação
    rb_prod QRadioButton Seleciona ambiente de Produção
    tb_consultar_sinistro QTableWidget Exibe os resultados da consulta
    btn_consultar_sinistro QPushButton Dispara a consulta à API
    btn_limpar_parametro_sinistro QPushButton Limpa campos e tabela
    btn_gerar_relatorios_sinistro QPushButton Exporta resultado em Excel

    Exclusividade entre Bilhete e CPF

    Os campos le_bilhete_sinistro e le_cpf_sinistro são mutuamente exclusivos: ao preencher um, o outro é automaticamente desabilitado. Isso garante que apenas um filtro seja enviado à API por vez.

    🔄 Fluxo de Consulta

    flowchart TD
    A[Usuário preenche Bilhete ou CPF] --> B{Campo preenchido?}
    B -->|Não| Z1[Aviso: preencha um campo]
    B -->|Sim| C{Ambiente selecionado?}
    C -->|Não| Z2[Aviso: selecione o ambiente]
    C -->|Sim| D[Desabilita btn_consultar]
    D --> E[Exibe QProgressDialog]
    E --> F[Inicia ConsultaApoliceThread]
    F --> G{Resultado?}
    G -->|Sucesso| H[consulta_sucesso: exibe na tabela]
    G -->|Erro| I[consulta_erro: exibe mensagem]
    H --> J[Habilita btn_gerar_relatorios]
    H --> K[Grava log no banco]

    📋 Métodos Principais

    consultar()

    Valida os campos, inicializa o QProgressDialog e dispara a ConsultaApoliceThread.

    Parâmetro / Detalhe Valor
    Filtro aceito nr_bilhete ou cpf_cnpj
    Ambientes homolog / prod
    Thread iniciada ConsultaApoliceThread(bilhete, cpf, ambiente) 
    self.thread_consulta = ConsultaApoliceThread(bilhete, cpf, ambiente)
self.thread_consulta.finished.connect(self.consulta_sucesso)
self.thread_consulta.erro.connect(self.consulta_erro)
self.thread_consulta.progresso.connect(self.progresso)
self.thread_consulta.start()
    consulta_sucesso(df)

    Chamada pelo sinal finished da thread. Popula a tabela e habilita o botão de exportação.

    Ação Descrição
    Fecha QProgressDialog Remove o bloqueio da UI
    exibir_na_tabela(df) Renderiza dados no QTableWidget
    Registra log Chama database.inserir_log_sistema_relatorio()
    Exibe mensagem Informa quantidade de registros encontrados
    consulta_erro(mensagem)

    Chamada pelo sinal erro da thread. Fecha o dialog e exibe a mensagem de erro ao usuário.

    def consulta_erro(self, mensagem: str):
    if self.progress_dialog:
        self.progress_dialog.close()
    self.main_window.mostrar_mensagem_customizada("critical", "Erro na Consulta", mensagem)
    self.ui.btn_consultar_sinistro.setEnabled(True)
    gerar_relatorio()

    Exporta o df_atual para um arquivo Excel na pasta Downloads do usuário.

    Detalhe Valor
    Formato do nome relatorio_consulta_item_{tipo}_{id}_{timestamp}.xlsx
    Destino ~/Downloads/{nome_pasta}/{nome_arquivo}
    Função de exportação consulta_apolice_api.exportar_relatorio_excel()

    Pasta individual por exportação

    Cada relatório é salvo em uma subpasta exclusiva com o mesmo nome base do arquivo, facilitando a organização no sistema de arquivos.

    limpar()

    Reseta todo o estado da página para o estado inicial:

    • Para a thread caso esteja em execução
    • Fecha o QProgressDialog se aberto
    • Limpa os QLineEdit e reabilita ambos
    • Desmarca e desabilita os QRadioButton
    • Zera o QTableWidget e o df_atual
    • Desabilita btn_gerar_relatorios_sinistro e btn_limpar_parametro_sinistro

    🧵 Thread: ConsultaApoliceThread

    Sinal Tipo Descrição
    finished pd.DataFrame Emitido com o DataFrame de resultados ao concluir
    erro str Emitido com a mensagem de erro em caso de falha
    progresso int Emitido com valores 25, 75 e 100 para atualizar o label do progress dialog
