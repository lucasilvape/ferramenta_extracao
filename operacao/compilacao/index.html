    📦 Compilação (Build)

    Este guia detalha o processo de geração do executável (.exe) para distribuição aos usuários finais.

    🛠️ Ferramenta: PyInstaller

    O projeto utiliza PyInstaller para empacotar a aplicação Python em um executável Windows standalone.

    Por que PyInstaller?

    • Gera um único arquivo .exe (sem necessidade de instalar Python)
    • Suporte nativo a PySide6/Qt
    • Configuração flexível via arquivo .spec

    📄 Arquivo de Configuração: build.spec

    O arquivo build.spec contém todas as configurações do build.

    build.spec
    # -*- mode: python ; coding: utf-8 -*-
from PyInstaller.utils.hooks import collect_submodules, collect_data_files

VERSAO_INTERFACE = "1.0.1"
NOME_EXECUTAVEL = f"Sistema de Relatorios v{VERSAO_INTERFACE}"

a = Analysis(
    ['src/main.py'],              # Arquivo de entrada
    pathex=['src'],               # Caminhos de busca
    datas=[
        ('assets', 'assets'),     # Inclui pasta de assets
    ] + collect_data_files('PySide6'),
    hiddenimports=[
        # Módulos do projeto
        'core', 'core.database', 'core.reports', 'core.conciliacao_cyber',
        'ui', 'ui.main_window', 'ui.icons_rc',

        # Bibliotecas externas
        'PySide6', 'PySide6.QtCore', 'PySide6.QtGui', 'PySide6.QtWidgets',
        'pandas', 'openpyxl', 'xlsxwriter',
        'pyodbc', 'psycopg2', 'dotenv', 'tqdm',
    ] + collect_submodules('PySide6'),
)

exe = EXE(
    pyz, a.scripts, a.binaries, a.zipfiles, a.datas, [],
    name=NOME_EXECUTAVEL,
    debug=False,
    console=True,                 # Mostra janela CMD (para debug)
    icon='assets/icons/logo_kovr2.png',
)

    🚀 Passo a Passo: Gerar o Executável

    1. Pré-requisitos

    Ambiente Virtual

    Certifique-se de estar no ambiente virtual do projeto.

    # Ativar ambiente virtual
.\venv\Scripts\activate

# Verificar se PyInstaller está instalado
pip show pyinstaller

    2. Executar o Build

    Comando de build
    pyinstaller build.spec

    3. Aguardar Compilação

    O processo pode levar 2-5 minutos dependendo do hardware.

    flowchart LR
    A[pyinstaller build.spec] --> B[Análise de dependências]
    B --> C[Coleta de arquivos]
    C --> D[Compressão UPX]
    D --> E[Geração do .exe]
    E --> F[dist/Sistema de Relatorios v1.0.1.exe]

    4. Localizar o Executável

    📁 FERRAMENTA_DE_RELATORIOS/
├── 📁 dist/
│   └── 🎯 Sistema de Relatorios v1.0.1.exe  ← AQUI
├── 📁 build/
│   └── ... (arquivos temporários)
└── ...

    📋 Configurações Importantes

    Hidden Imports

    Módulos que não são detectados automaticamente pelo PyInstaller.

    Módulo Motivo
    psycopg2 Driver PostgreSQL com bindings C
    pyodbc Driver ODBC com bindings C
    PySide6.* Widgets Qt carregados dinamicamente
    core.* Módulos internos do projeto

    Data Files

    Arquivos que precisam ser incluídos no executável.

    datas=[
    ('assets', 'assets'),  # Toda a pasta assets
] + collect_data_files('PySide6'),  # Plugins Qt

    Console Mode

    console=True,  # Mostra janela CMD

    Para produção

    Altere para console=False para esconder a janela do CMD no executável final.

    🔄 Atualizar Versão

    Para gerar uma nova versão:

    1. Atualize a versão no build.spec: 

      VERSAO_INTERFACE = "1.0.2"

    2. Atualize a versão no src/main.py: 

      VERSAO_INTERFACE = "1.0.2"

    3. Insira a versão no banco (para auto-update): 

      database.inserir_versao_sistema('1.0.2')

    4. Gere o novo executável: 

      pyinstaller build.spec

    5. Distribua o arquivo para a pasta de rede: 

      \\AWSSEG130006\ferramenta_extracao\

    🐛 Troubleshooting

    Erro: ModuleNotFoundError

    Problema: Módulo não encontrado ao executar o .exe.

    Solução: Adicione o módulo em hiddenimports:

    hiddenimports=[
    # ...
    'nome_do_modulo',
]

    Erro: Failed to execute script

    Problema: Crash ao iniciar o executável.

    Solução: Compile com console=True e verifique o erro no CMD.

    Executável muito grande (>200MB)

    Problema: Tamanho excessivo do arquivo.

    Solução: Verifique se não há bibliotecas desnecessárias instaladas no venv.

    pip list  # Lista todas as bibliotecas
